# Documentation

Guidelines for writing documentation for NotAlterra.

## Style

- End-user README: clear installation instructions, feature list, privacy statement
- No build instructions in end-user docs — those go in scripts/ or skills/
- Be factual and humorous where appropriate
- Use tables for comparison (installer variants, dependencies)

## Files

| File | Audience | Content |
|------|----------|---------|
| README.md | End users | Install, features, safety, backup format |
| docs/DECISIONS.md | Developers | ADRs and technical rationale |
| docs/KNOWN_ISSUES.md | Users/devs | Bugs and workarounds |
| docs/AUDIT.md | Security review | Attack surface, data storage |
| docs/GOVERNANCE.md | Contributors | Release process, licensing |
| docs/SIDEBAR.md | Developers | Sidebar video attribution |
